Práctica 1 – Uso de Git y GitHub

Explicación de comandos:

1. git clone

El comando `git clone` se utiliza para copiar un repositorio remoto en un entorno de trabajo local.
En esta práctica, el repositorio se clonó automáticamente al crear un GitHub Codespace a partir del repositorio fork.
Al pasar del fork al Codespace (workspace), GitHub realiza internamente el `git clone` y deja el repositorio disponible en la ruta `/workspaces/p1`.

Por este motivo, no fue necesario ejecutar el comando manualmente, ya que el resultado es el mismo que si se hubiera hecho desde un entorno vacío.

--------------------------------------------------

2. git status

Este comando se utiliza para ver el estado del repositorio, indicando en qué rama se está trabajando y si existen cambios pendientes.

Primer uso:
Se utilizó al inicio para comprobar que el repositorio estaba en la rama principal (`main`), sincronizado con GitHub y sin cambios pendientes.
Salida obtenida:
On branch main
Your branch is up to date with 'origin/main'.
nothing to commit, working tree clean

Segundo uso:
Se utilizó después de crear una nueva rama para comprobar que el cambio de rama se había realizado correctamente.
Salida obtenida:
On branch feature/1
nothing to commit, working tree clean

--------------------------------------------------

3. git add

Se emplea para indicar a Git qué archivos se quieren guardar en el siguiente commit.
Con él añadimos el archivo `git.txt` al área de preparación antes de confirmar los cambios.

Comando ejecutado:
git add git.txt

--------------------------------------------------

4. git commit

Se usa para guardar de forma definitiva los cambios preparados en el repositorio local.
Se utilizó para confirmar la creación del archivo `git.txt` con la explicación de los comandos Git.

Comando ejecutado:
git commit -m "Explicación de comandos Git"

Salida obtenida:
[main 6962812] Explicación de comandos Git
 1 file changed, 13 insertions(+)
 create mode 100644 git.txt

--------------------------------------------------

5. git push

Se utiliza para enviar los cambios guardados en el repositorio local al repositorio remoto en GitHub.
En esta práctica se utilizó para subir el archivo `git.txt` al repositorio fork.

Comando ejecutado:
git push origin main

--------------------------------------------------

6. git checkout

Se utiliza para cambiar de rama o crear una nueva rama.
En esta práctica se creó una nueva rama para probar el comando y después se volvió a la rama principal.

Comandos ejecutados:
git checkout -b feature/1
git checkout main

Estos comandos permiten trabajar en una rama distinta sin modificar la rama principal.
